# VisionTrack — Local Video Analytics Platform

A complete MVP for uploading CCTV footage, running object detection/tracking, and downloading structured Excel reports. Designed to run entirely on a local workstation (Lenovo P360 + Axelera AI accelerator) with zero cloud dependencies.

---

## Architecture

```

│   React Frontend    │  HTTP  │   FastAPI Backend     │
│   (Vite · :3000)    │◄──────►│   (Uvicorn · :8000)  │
```

## Quick Start

### Prerequisites

- **Python 3.10+** with pip
- **Node.js 18+** with npm

### 1. Clone / extract the project

```bash
cd visiontrack
```

### 2. Set up the backend

```bash
# Create virtual environment (recommended)
python3 -m venv .venv
source .venv/bin/activate        # Linux/macOS
# .venv\Scripts\activate         # Windows

# Install dependencies
pip install -r backend/requirements.txt
```

### 3. Start the backend

```bash
uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
```

The API is now live at **http://localhost:8000**. Visit http://localhost:8000/docs for interactive Swagger docs.

### 4. Set up the frontend (new terminal)

```bash
cd frontend
npm install
```

### 5. Start the frontend

```bash
npm run dev
```

Open **http://localhost:3000** in your browser.

## Usage

1. **Upload** — Drag and drop a video file (MP4, AVI, MOV, MKV) onto the upload zone, or click to browse.
2. **Preview** — Hover over the video thumbnail to preview before uploading.
3. **Process** — Click "Upload & Process". The file uploads with a progress bar, then processing starts automatically.
4. **Monitor** — The jobs table updates every 2.5 seconds. Watch the status change from Processing → Complete.
5. **Download** — Click the green "↓ Report" button to download the Excel analytics report.

| Method   | Endpoint              | Description               |
|----------|-----------------------|---------------------------|

| `GET`    | `/download/{job_id}`  | Download Excel report     |
| `DELETE` | `/jobs/{job_id}`      | Delete a job and files    |

## Processing Pipeline

The `process_video.py` script currently runs a **mock YOLO pipeline** that:

1. Reads video file metadata (size, estimated duration)
2. Simulates frame-by-frame object detection with realistic class distributions
3. Tracks objects across frames with unique IDs
4. Aggregates statistics per zone and per class
5. Generates a multi-sheet Excel report with:
   - **Summary** — Video metadata and high-level counts
   - **Zone Summary** — Objects per zone with bar chart
   - **Class Breakdown** — Detection counts per object type
   - **Tracks** — Detailed per-track information
   - **Raw Detections** — Frame-level detection data (sampled)

### Replacing with Real YOLO

To use real detection, replace the `run_detection()` function in `process_video.py`:

```python
from ultralytics import YOLO

model = YOLO("yolov8n.pt")  # or Axelera-compiled model

def run_detection(video_path, num_frames):
    detections = []
    results = model.track(source=video_path, stream=True, persist=True)
    for frame_idx, result in enumerate(results):
        for box in result.boxes:
            detections.append({
                "frame": frame_idx,
                "timestamp_s": frame_idx / 25.0,
                "track_id": int(box.id) if box.id is not None else -1,
                "class": model.names[int(box.cls)],
                "zone": classify_zone(box.xyxy),  # your zone logic
                "confidence": float(box.conf),
                "bbox_x": int(box.xyxy[0][0]),
                "bbox_y": int(box.xyxy[0][1]),
                "bbox_w": int(box.xyxy[0][2] - box.xyxy[0][0]),
                "bbox_h": int(box.xyxy[0][3] - box.xyxy[0][1]),
            })
    return detections
```

## Configuration

Key settings in `backend/main.py`:

- `MAX_FILE_SIZE_MB = 2048` — Maximum upload size (default 2 GB)
- `ALLOWED_EXTENSIONS` — Supported video formats
- Polling interval in frontend: `2500ms` (adjustable in `App.jsx`)

## Scaling Notes

This MVP is structured for easy extension:

- **Database**: Replace `jobs.json` with SQLite/PostgreSQL via SQLAlchemy
- **Queue**: Replace `BackgroundTasks` with Celery + Redis for multi-worker processing
- **Storage**: Swap local paths with S3-compatible storage (MinIO for local, AWS S3 for cloud)
- **Auth**: Add JWT authentication middleware to FastAPI
- **Real-time**: Replace polling with WebSocket updates
- **Axelera Integration**: Load Axelera-compiled YOLO model via their SDK in `process_video.py`

## Troubleshooting

| Issue | Fix |
|-------|-----|
| CORS errors | Backend must be on port 8000; check CORS middleware in `main.py` |
| Upload fails | Check file extension is allowed; verify file < 2 GB |
| Processing stuck | Check terminal running the backend for error logs |
| Report missing | Verify `/reports/` directory exists and is writable |
| Port in use | Kill existing process: `lsof -ti:8000 \| xargs kill` |
